# Unlist

Used car listings on Spinny and Cars24 bury inspection faults in fine print most buyers never read. Unlist surfaces every fault, ranks it by severity, and gives you a plain-English summary before you walk into the showroom.

## What it does

Paste a Spinny or Cars24 listing URL. Get back:

- Every inspection fault ranked by severity (1–5), not just Spinny's binary critical/non-critical
- Separate buckets for unresolved issues, restored parts, and replaced parts
- A test drive checklist of things to physically verify at the showroom
- Tyre life remaining per wheel (Cars24)
- A plain-English summary with negotiation advice, generated by Gemini
- A shareable permalink to the report, valid for 7 days

## How it works

**Spinny** — listing URLs expose a car ID which maps to two internal API endpoints: one for category-level scores and brand new parts, one for granular part-level fault data including criticality and status of condition per part.

**Cars24** — the website gates inspection reports behind an app download. The Cars24 Android app was reverse engineered by intercepting SSL-pinned traffic via APK repackaging and Charles Proxy, exposing an internal API endpoint that returns full inspection data including tyre life percentages and part-level faults.

**ML severity scoring** — Spinny's inspection data marks faults as `is_critical: true/false` but treats "Check Engine error code" and "mildly faded mirror cover" as equally critical. A Random Forest classifier trained on a manually labelled dataset of 1,900 faults across 100 listings predicts a severity score from 1–5 per fault, enabling proper ranking.

**Caching** — analyse results are cached in Redis for 30 days. Repeated lookups of the same listing skip the Gemini call entirely and return instantly.

## Tech stack

- **Backend** — FastAPI, Python. Deployed on Render.
- **ML** — scikit-learn (Random Forest), trained on 1,900 manually labelled fault records. pandas for data processing.
- **LLM** — Gemini 3.1 Flash Lite for plain-English report summaries.
- **Cache** — Redis (Render Key Value) for analyse result caching and shareable report storage.
- **Frontend** — React (in progress)

## API

`POST /analyse`

```json
{ "url": "https://www.spinny.com/buy-used-cars/..." }
```

Returns structured fault report with severity scores, checklist, tyre life, and summary.

`POST /compare`

```json
{ "urls": ["https://www.spinny.com/...", "https://www.cars24.com/..."] }
```

Side-by-side comparison of 2–4 listings. Supports mixing Spinny and Cars24 URLs.

`POST /share`

```json
{ "report": { ...analyse or compare response... } }
```

Persists a report and returns a short ID. Reports expire after 7 days.

`GET /report/{id}`

Returns a previously shared report by ID.

---

Supports Spinny listings and Cars24 owned stock. Cars24 direct seller listings are not supported as their inspection reports are behind a paywall.
