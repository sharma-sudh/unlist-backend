# Unlist

Used car listings on Spinny and Cars24 bury inspection faults in fine print most buyers never read. Unlist surfaces every fault, ranks it by severity, and gives you a plain-English summary before you walk into the showroom.

## What it does

Paste a Spinny or Cars24 listing URL. Get back:

- Every inspection fault ranked by severity (1–5), not just Spinny's binary critical/non-critical
- Separate buckets for unresolved issues, restored parts, and replaced parts
- A test drive checklist of things to physically verify at the showroom
- Tyre life remaining per wheel (Cars24)
- A plain-English summary with negotiation advice, generated by Gemini
- Side-by-side comparison of 2–4 listings with a Gemini-generated verdict
- Shareable report permalinks, valid for 7 days
- Saved reports tied to a user account, accessible across sessions

## How it works

### Spinny

Listing URLs expose a numeric car ID. This maps to two unauthenticated internal REST endpoints — one returning category-level scores and brand new parts, one returning granular part-level fault data with `name`, `soc`, `is_critical`, and `condition` fields per part. No scraping required.

### Cars24 — SSL Pinning Bypass

Cars24 is the technically interesting one. The website gates full inspection reports behind an app download wall — there is no web-accessible equivalent. To get the data, the Cars24 Android APK was reverse engineered:

1. **Decompiled the APK** to identify the certificate pinning implementation
2. **Repackaged the APK** with the pinning checks removed
3. **Proxied traffic through Charles Proxy** with the modified app to intercept HTTPS requests
4. **Identified the internal API endpoint** (`car-catalog-gateway-in.c24.tech/detail/v1/`) and the full request structure — headers, Android app version strings, and JSON body format
5. **Replicated the request** server-side using the intercepted headers to impersonate the Android app

This exposed the complete inspection report including tyre life percentages per wheel, part-level fault data, and repaint detection — none of which is accessible through any official Cars24 web interface.

### ML Severity Scoring

Spinny marks faults as `is_critical: true/false` but applies the same binary label to "Check Engine error code" and "mildly faded mirror cover". A Random Forest classifier trained on a manually labelled dataset of 1,900 faults across 100 listings predicts a severity score from 1–5 per fault across 10 categories (brake, engine, transmission, suspension, steering, chassis, glass, body, electrical, accessory), enabling proper ranking by actual risk.

### Caching

Analyse results are cached in Redis for 30 days. Repeated lookups of the same listing skip scraping and the Gemini call entirely and return instantly.

## Tech stack

- **Backend** — FastAPI, Python. Deployed on Render.
- **ML** — scikit-learn (Random Forest), trained on 1,900 manually labelled fault records. pandas for data processing.
- **LLM** — Gemini 3.1 Flash Lite for plain-English report summaries and compare verdicts.
- **Cache** — Redis (Render Key Value) for analyse result caching and shareable report storage.
- **Database** — PostgreSQL (Neon) for user accounts and saved reports.
- **Auth** — Google OAuth 2.0 + email/password, both issuing 30-day JWTs.
- **Frontend** — React (in progress)

## API

### Public

`POST /analyse`
```json
{ "url": "https://www.spinny.com/buy-used-cars/..." }
```
Returns a structured fault report with severity scores, checklist, tyre life, and Gemini summary.

`POST /compare`
```json
{ "urls": ["https://www.spinny.com/...", "https://www.cars24.com/..."] }
```
Side-by-side comparison of 2–4 listings with a Gemini verdict. Supports mixing Spinny and Cars24 URLs.

`POST /share`
```json
{ "report": { ...analyse or compare response... } }
```
Persists a report anonymously and returns a short ID. Expires after 7 days.

`GET /report/{id}` — Returns a previously shared report by ID.

### Auth

`GET /auth/login` — Redirects to Google consent screen.

`GET /auth/callback` — Google OAuth callback. Returns `{ token, user }`.

`POST /auth/register`
```json
{ "email": "user@example.com", "password": "Password1", "name": "Name" }
```

`POST /auth/login`
```json
{ "email": "user@example.com", "password": "Password1" }
```

`GET /auth/me` — Returns the authenticated user's claims. Requires `Authorization: Bearer <token>`.

### Saved reports (protected)

All endpoints require `Authorization: Bearer <token>`.

`POST /saved` — Save a report to the user's account.

`GET /saved` — List all saved reports for the authenticated user.

`DELETE /saved/{id}` — Delete a saved report.

---

Supports Spinny listings and Cars24 owned stock. Cars24 direct seller listings are not supported as their inspection reports are behind a paywall.
